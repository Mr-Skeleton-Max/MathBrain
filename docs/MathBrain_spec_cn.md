## 1. 组件一：HashRetina — 哈希视网膜编码

### 1.1 定义与设计理由

HashRetina 是系统的**前端编码器**，将词的字符序列映射到稀疏的虚拟槽位空间 $K$。它是零参数、确定性的，通过 N-gram 哈希捕捉字形结构，让形态相似的词共享部分槽位，实现字形级别的泛化。

系统的所有后续组件（Q、B、A）均在槽位空间 $K$ 上运行，而非词表空间 $V$。

### 1.2 编码流程

对词 $w$ 加边界符后提取所有 N-gram（默认 trigram, $n=3$）：

```
word = "apple"
padded = "^apple$"
trigrams = ["^ap", "app", "ppl", "ple", "le$"]
```

统计频次并哈希到槽位：

$$\text{slot\_counts}[j] = \sum_{\text{ng}: \text{hash}(\text{ng}) = j} \text{count}[\text{ng}]$$

返回频次字典 `Dict[slot_id, count]`，作为 Q 更新的刺激信号。

### 1.3 槽位共享与字形泛化

| 词对 | 共享 trigrams | Jaccard 相似度 |
|------|---------------|----------------|
| apple / apples | `^ap`, `app`, `ppl`, `ple` | ~0.67 |
| cat / cats | `^ca`, `cat` | ~0.40 |
| run / running | `^ru`, `run` | ~0.25 |

不相关的词在 $K=65536$ 下几乎零共享。学习了 "cat" 的知识可部分迁移到 "cats"。

### 1.4 参数

| 参数 | 推荐值 | 含义 |
|------|--------|------|
| $K$ | 65536 | 虚拟槽位数 |
| $n$ | 3 (trigram) | N-gram 大小 |

详细的编码变体（V1/V2）、实验验证见附录 F。

## 2. 组件二：Q — 语境状态编码

### 2.1 定义与设计理由

$Q \in \mathbb{R}^{K \times N}$（稀疏存储）是系统的**语境状态**。它用 $N$ 个不同衰减率的指数移动平均（EMA）通道编码每个槽位的近期激活历史，等价于该历史的**实 Laplace 变换**在 $N$ 个采样点处的值。

$Q$ 不存储原始序列，而是将可变长度的时序信息压缩为固定大小的状态矩阵。每一列 $Q[:, k]$ 对应一个时间尺度——快通道（$k=0$）只保留最近几步，慢通道（$k=N{-}1$）保留长程记忆。

### 2.2 时间尺度参数

定义窗口宽度 $w_k$（"在该尺度上有效的回看距离"）：

$$w_k = w_{\text{min}} \cdot \left(\frac{w_{\text{max}}}{w_{\text{min}}}\right)^{k/(N-1)}, \quad k = 0, \ldots, N-1$$

对数等分确保各尺度覆盖从短程到长程的完整范围。保留率和衰减率：

$$\rho_k = 1 - \frac{1}{w_k}, \quad \beta_k = 1 - \rho_k = \frac{1}{w_k}$$

**推荐值**：$N=4$，$w_{\text{min}} = 2$，$w_{\text{max}} = 100$，给出 $\rho \approx [0.50, 0.86, 0.95, 0.99]$。实际代码默认使用 $\rho = [0.3, 0.6, 0.85, 0.95]$。

### 2.3 Q 更新规则

收到当前词 $w_\text{cur}$ 时，先通过 HashRetina 编码为槽位频次 $\text{slot\_counts} = \text{HashRetina}(w_\text{cur})$：

**步骤 1a — 全局衰减**：对所有活跃槽位 $i, k$：

$$Q[i, k] \leftarrow \rho_k \cdot Q[i, k]$$

**步骤 1b — 槽位写入**（频次加权）：

$$Q[j, k] \leftarrow Q[j, k] + \text{count}[j] \quad \forall j \in \text{slot\_counts}, \forall k$$

**步骤 1c — 稀疏截断**：$Q[i,k] \leftarrow 0$ if $Q[i,k] < \epsilon_Q$。

**关于频次加权写入**：写入 `count[j]`（而非固定常数 1）让高频 N-gram 获得更强的激活，反映字形结构的重要性。各通道仍编码同一个出现函数的 Laplace 变换，保持数学统一性（详见 §2.4）。

### 2.4 Q 的数学性质

**单次出现的衰减**。若 token $i$ 在当前位置之前第 $\delta$ 步出现过一次：

$$Q[i,k] = \rho_k^{\delta}$$

这是关于距离 $\delta$ 的指数衰减函数。

**多次出现的叠加**。若 token $i$ 在之前第 $\delta_1, \delta_2, \ldots, \delta_M$ 步出现过：

$$Q[i,k] = \sum_{m=1}^{M} \rho_k^{\delta_m}$$

**Laplace 变换解释**。令 $b(\delta) \in \{0, 1\}$ 为 token $i$ 在距离 $\delta$ 处的出现指示函数，$s_k = -\ln(\rho_k)$：

$$Q[i,k] = \sum_{\delta} b(\delta) \cdot e^{-s_k \cdot \delta} = \tilde{b}(s_k)$$

$N$ 个通道 $Q[i, 0], \ldots, Q[i, N{-}1]$ 共同构成 $b(\delta)$ 的实 Laplace 变换在 $N$ 个不同 $s$ 值处的采样。这一结构与 Marc Howard 实验室的 TILT 框架（Laplace Neural Manifold）高度一致——大脑中的 Laplace cells 是一组具有不同衰减率的漏积分器，其群体活动编码刺激历史的 Laplace 变换。

**唯一性**。对固定的 $\rho_k \in (0,1)$，不同的距离集合 $\{\delta_m\}$ 产生不同的 Q 值。$\{\rho_k^n\}_{n \geq 0}$ 是严格递减正数序列，其子集和具有唯一分解性。多尺度 $N$ 个通道提供数值冗余，增强有限精度下的鲁棒性。

### 2.5 示例

$V=5$, $N=3$, $\rho = [0.50, 0.80, 0.95]$。序列 `[A, B, C, A, D]`，读完后：

| token | $k=0$ ($\rho=0.50$) | $k=1$ ($\rho=0.80$) | $k=2$ ($\rho=0.95$) |
|-------|---------------------|---------------------|---------------------|
| A | $0.50^1 + 0.50^4 = 0.5625$ | $0.80^1 + 0.80^4 = 1.210$ | $0.95^1 + 0.95^4 = 1.764$ |
| B | $0.50^3 = 0.125$ | $0.80^3 = 0.512$ | $0.95^3 = 0.857$ |
| C | $0.50^2 = 0.250$ | $0.80^2 = 0.640$ | $0.95^2 = 0.903$ |
| D | $0.50^0 = 1.000$ | $0.80^0 = 1.000$ | $0.95^0 = 1.000$ |

A 在 $k=0$ 上几乎只保留最近出现（$\delta=1$），在 $k=2$ 上两次出现的贡献接近等权。D 刚出现，所有通道值为 1。

## 3. 组件三：$\varphi$ — Cosine Chaos 距离特征编码

### 3.1 动机

Q 编码在 Laplace 域，直接用 Q 做点积无法有效利用 EMA 编码的距离信息。早期设计使用逆 Laplace 投影 $L^{-1}$（见附录 E.1），但在长上下文中相邻位置的 $\varphi$ 高度相似（余弦 0.96–0.99），局部区分度不足。

Cosine Chaos 编码通过非线性折叠解决此问题：近处 token 变化产生高混沌（强区分），远处 token 变化产生低混沌（强泛化）。

### 3.2 编码公式

$$\varphi = \cos(E_\text{proj} \cdot (\boldsymbol{\alpha} \odot Q))$$

其中：
- $E_\text{proj} \in \mathbb{R}^{D \times N}$：固定随机投影矩阵（$\mathcal{N}(0, \sigma^2)$，$\sigma = 0.5$，seed=42）
- $\boldsymbol{\alpha} \in \mathbb{R}^N$：反向缩放的混沌强度参数
- $\odot$：逐元素乘法
- $\cos$：逐元素余弦函数

### 3.3 反向缩放 $\alpha$（多尺度混沌控制）

$$\alpha_k = \frac{(1-\rho_k)}{\text{mean}(1-\rho)} \cdot \alpha_\text{base}$$

| 尺度 | $\rho_k$ | $\alpha_k$ | 作用 |
|------|----------|------------|------|
| 0 (快) | 0.30 | 大 | 高混沌 → 近邻精确区分 |
| 1 | 0.60 | 中 | 中等混沌 |
| 2 | 0.85 | 小 | 低混沌 |
| 3 (慢) | 0.95 | 极小 | 极低混沌 → 远程语境泛化 |

反向缩放 $\alpha_k \propto (1-\rho_k)$ 完美抵消各尺度的稳态能量差异（$Q_{k,\infty} = 1/(1-\rho_k)$），确保各尺度的相空间体积被钳制在同一数量级。

### 3.4 核心性质

**近邻高区分 + 远程高泛化**：EMA 衰减提供范数梯度，Cosine 将范数梯度转化为混沌梯度：
- 近处 token 变化 → $\Delta Q$ 大 → 触发 $\cos$ 折叠 → $\varphi$ 变化剧烈 → 区分
- 远处 token 变化 → $\Delta Q$ 小 → 不触发折叠 → $\varphi$ 变化平滑 → 泛化

**语义饱和**：重复输入相同 token 时 $\varphi$ 趋于饱和（相似度 → 1.0），自动过滤冗余信息。输入新 token 时仍能区分（相似度 ≈ -0.55）。

**范数有界**：$\cos$ 将输出限制在 $[-1, 1]$，消除高频词累积范数淹没问题。

**满秩特征**：$\varphi$ 均匀散布在超球面上，协方差矩阵有效秩接近 $D$（实测 7.95/8），条件数 ≈ 1.5，极大改善 Sleep 阶段 CP 分解的收敛性。

### 3.5 参数

| 参数 | 推荐值 | 含义 |
|------|--------|------|
| $D$ | 64 | 距离特征维度 |
| $\sigma$ | 0.5 | $E_\text{proj}$ 初始化标准差 |
| $\alpha_\text{base}$ | $\pi/(2\sigma\sqrt{D})$ 或经验值 | 基础混沌强度 |
| 缩放方式 | 反向 $\propto (1-\rho_k)$ | 快尺度大、慢尺度小 |

### 3.6 与神经科学的对应

| MathBrain | 大脑 |
|-----------|------|
| Q（多衰减率 EMA） | Laplace cells（漏积分器群体） |
| $E_\text{proj}$（固定随机投影） | 前馈随机连接 |
| 多尺度 $\alpha$（反向缩放） | 网格细胞（背侧密集/腹侧稀疏） |
| $\varphi$（Cosine Chaos 输出） | Timing cells（时间感受野） |

## 4. 组件四：B — 连接张量（在线精确记忆）

### 4.1 定义与设计理由

$B \in \mathbb{R}^{K \times K \times D}$ 是系统的**短期精确记忆**。$B[v, i, m]$ 编码的是："当槽位 $i$ 在距离 $\tau_m$ 附近出现时，对 target 槽位 $v$ 的预测贡献"。

B 的角色是**海马体**：快速在线写入、精确但有限容量、需要通过 Sleep 巩固到 A。

### 4.2 B 通道预测

$$l_B[v] = \sum_{i \in \text{active}} \sum_{m=0}^{D-1} B[v, i, m] \cdot \hat{\varphi}[i, m]$$

其中 $\hat{\varphi}[i] = \varphi[i] / \|\varphi[i]\|$ 为归一化距离特征，active = $\{i : \|Q[i,:]\| > \theta_Q\}$。矩阵形式：

$$\mathbf{l}_B = \sum_{i \in \text{active}} B[:, i, :] \cdot \hat{\varphi}[i, :] \in \mathbb{R}^K$$

**归一化的作用**：使 $\hat{\varphi} \cdot B[v,i,:]$ 可解释为 $[0,1]$ 区间的投影系数（连接强度），与能量守恒写入规则（§4.3）对齐。不同编码维度下行为一致。

### 4.3 B 的写入（统一突触能量守恒）

B 的写入基于**能量守恒**原则：每条 $(v, i)$ 连接的总投影强度趋近 1，B 作为双向校准器补偿 A 的偏差。

**φ 归一化**：写入和预测均使用归一化距离特征：

$$\hat{\varphi}[i] = \frac{\varphi[i]}{\|\varphi[i]\|}$$

**Per-connection 强度计算**：

$$s_B = \hat{\varphi}[i] \cdot B[v, i, :], \quad s_A = \hat{\varphi}[i] \cdot \hat{A}[v, i, :]$$

其中 $\hat{A}[v,i,:]$ 由 CP 因子重构（§4.1）。A 无知识时 $s_A = 0$。

**双向能量写入**（对实际 next-token $v^*$，所有活跃 source $i$）：

$$g = 1 - (s_A + s_B)$$

$$\Delta B[v^*, i, :] = \eta \cdot g \cdot \hat{\varphi}[i]$$

- $g > 0$（A 欠冲）：B 正向补偿，增强连接
- $g < 0$（A 过冲）：B 负向抑制，压制连接
- $g = 0$（总强度 = 1）：B 停写，能量平衡

**稀疏衰减**（惰性执行）：$B[v,i,:] \leftarrow (1-\lambda_B)^{\Delta t} \cdot B[v,i,:]$

衰减仅在条目被访问时（预测或写入）执行，未访问的条目保持原值。$\Delta t$ 为自上次更新以来的步数。这种惰性策略避免了遍历所有稀疏条目的开销。

**稀疏裁剪**：$B[v,i,:] \leftarrow 0$ if $\|B[v,i,:]\| < \epsilon_\text{prune}$。

### 4.4 能量守恒的数学性质

**压缩映射**。令 $S = s_A + s_B$ 为总强度，单步更新后：

$$S_\text{new} = S + \eta \cdot (1 - S) = (1-\eta) \cdot S + \eta$$

这是以 1 为不动点、$(1-\eta)$ 为压缩率的仿射映射。无论 $S$ 从任何初值出发，都单调趋近 1，**数学上不可能振荡**。

**双向校准解决 CP 过冲**。A 的 CP 分解是全局优化，共享参数可能导致部分连接 $s_A > 1$（过冲）。单向写入（$g = \max(0, 1-S)$）无法修正过冲，导致不对称漂移：欠冲被修正，过冲被锁死。双向写入允许 B 写入负修正，通过 Sleep 累积反馈逐轮收敛 A 的过冲。

| 性质 | 双向能量守恒 | 单向能量守恒 |
|------|-------------|-------------|
| 欠冲修正 | ✓ $g > 0$ 正向补偿 | ✓ 同 |
| 过冲修正 | ✓ $g < 0$ 负向抑制 | ✗ $g = 0$ 锁死 |
| 收敛性 | 压缩映射，保证收敛 | 仅从下方收敛 |
| A 漂移控制 | Sleep 吸收负修正，逐轮收敛 | 过冲单调膨胀 |

### 4.5 示例

语境 `the cat`，A 已学到 $s_A(\text{eat}, \text{cat}) = 1.15$（CP 过冲），实际 next-token 为 eat：

$$g = 1 - (1.15 + 0) = -0.15$$

$$\Delta B[\text{eat}, \text{cat}, :] = 0.5 \times (-0.15) \times \hat{\varphi}[\text{cat}] = -0.075 \cdot \hat{\varphi}[\text{cat}]$$

B 写入负修正。Sleep 时 $\text{target} = \hat{A}_\text{old} + B = A_\text{old} - 0.075 \cdot \hat{\varphi}$，A 学到更小的向量，下轮过冲减少。

## 5. 组件五：A — CP 分解知识库（长期泛化记忆）

### 5.1 定义与设计理由

A 是系统的**长期知识库**，通过 CP（CANDECOMP/PARAFAC）分解将 $K \times K \times D$ 的张量压缩到共享嵌入空间中。A 的角色是**新皮层**：缓慢巩固、高容量、支持跨槽位泛化。

$$\hat{A}[v, i, m] = \sum_{r=1}^{d} (E \cdot W_Q)[v, r] \cdot (E \cdot W_V)[i, r] \cdot P[m, r]$$

四个因子矩阵：

| 因子 | 形状 | 含义 |
|------|------|------|
| $E$ | $K \times d$ | 共享槽位嵌入（正交约束 $E^\top E \approx I$） |
| $W_Q$ | $d \times d$ | Target 角色投影 |
| $W_V$ | $d \times d$ | Source 角色投影 |
| $P$ | $D \times d$ | 距离基投影 |

总参数量 $\approx Kd + 2d^2 + Dd$。由于 $d$ 为常数级（384），A 的存储与 $K$ 线性相关，压缩比随 $K$ 增长。

### 5.2 共享嵌入的设计

$E$ 是所有槽位共享的基础嵌入。$W_Q$ 和 $W_V$ 将 $E$ 投影到不同的角色空间：

- **Target 角色** $A_Q = E \cdot W_Q$：编码"什么语境预测我"
- **Source 角色** $A_V = E \cdot W_V$：编码"我在语境中时预测什么"

双角色投影解耦了 target-side 和 source-side 的相似性结构：cat 和 dog 的 target 角色可能相似（都在 "the \_\_\_" 后出现），但 source 角色不同（cat 后跟 mouse，dog 后跟 ball）。

### 5.3 A 通道预测

$$l_A[v] = \sum_{i \in \text{active}} \sum_{m=0}^{D-1} \hat{A}[v, i, m] \cdot \hat{\varphi}[i, m]$$

展开 CP 结构（使用归一化 $\hat{\varphi}$，与 B 通道对齐）：

$$l_A[v] = (E \cdot W_Q)[v, :] \cdot \left(\sum_{i \in \text{active}} (E \cdot W_V)[i, :] \odot (P^\top \cdot \hat{\varphi}[i, :]) \right)$$

计算流程：
1. 对每个活跃 source $i$，计算 $\mathbf{s}_i = (E \cdot W_V)[i, :] \odot (P^\top \cdot \hat{\varphi}[i, :]) \in \mathbb{R}^d$
2. 汇总 $\mathbf{c} = \sum_i \mathbf{s}_i$（嵌入空间的上下文向量）
3. 对所有 target：$\mathbf{l}_A = (E \cdot W_Q) \cdot \mathbf{c} \in \mathbb{R}^K$

时间 $O(K \cdot d \cdot D + K \cdot d)$，其中 $K$ 为活跃 source 数。

### 5.4 泛化机制：投影空间

**关键实验发现**：零样本泛化不依赖嵌入空间 $E$ 中的显式聚类。即使 $\cos(E[\text{fox}], E[\text{cat}]) \approx 0.05$（几乎正交），CP 投影空间中同组动物的输出高度一致：

$$\cos(\hat{A}[\text{fox, eat}], \hat{A}[\text{cat, eat}]) \approx 0.70$$

泛化路径是 $E \cdot W_Q$ 和 $E \cdot W_V$ 的矩阵乘法——它从表面不相似的嵌入中提取语义共性。$W_Q, W_V$ 作为共享投影矩阵，将不同 token 在共享上下文中积累的统计规律性映射到同一 CP 方向。

### 5.5 正交约束

$E$ 受正交约束 $E^\top E \approx I$：

$$\mathcal{L}_\text{ortho} = \lambda_\text{ortho} \lVert E^\top E - I \rVert_F^2$$

**作用**：

1. **坐标系锁死**：$E$ 的列定义语义空间的坐标轴，正交约束禁止坐标轴旋转，旧知识的投影坐标不因新知识学习而漂移。
2. **知识路由分离**：学习新知识时优化器无法通过旋转 $E$ 走捷径，被迫通过调整 $W_Q, W_V$ 路由，对旧知识的干扰被结构性消除。
3. **相变行为**：即使 $\lambda_\text{ortho}$ 低至 0.01，$\lVert E^\top E - I \rVert_F$ 就从 8+ 骤降至 $\approx 0$。

### 5.6 A 与 B 的对比

| | B 通道 | A 通道 |
|---|---|---|
| 存储 | $K^2 D$ 稀疏条目 | $Kd + 2d^2 + Dd$ 参数 |
| 写入 | 在线能量守恒（毫秒级） | Sleep 巩固（离线优化） |
| 泛化 | 无（$B[\text{slot}_1]$ 与 $B[\text{slot}_2]$ 独立） | 有（共享 $E$ 和投影矩阵） |
| 角色 | 海马体：快速精确但有限 | 新皮层：缓慢泛化且持久 |

## 6. 组件六：Sleep — 伪排练增量巩固

### 6.1 核心思想

Sleep 将 B 中积累的新信息巩固到 A 中，然后清零 B 释放容量。巩固过程使用**伪排练**（Pseudorehearsal）防止灾难性遗忘，并附加正交约束和置信度掩码。

### 6.2 数学目标

$$\hat{A}_\text{new} \approx \hat{A}_\text{old} + B_\text{day}$$

$A$ 始终以 CP 因子 $(E, W_Q, W_V, P)$ 存储。Sleep 的目标是找到因子的最小调整，使 $\hat{A}_\text{new}$ 在 B 有信号的位置吸收新信息，同时在其他位置保持旧知识。

### 6.3 伪排练解法

训练时混合两种样本：

**新知识样本**：从 B 中取有信号的 $(v, i)$ 位置

$$\text{target}_\text{new}[v, i, :] = \hat{A}_\text{old}[v, i, :] + B[v, i, :]$$

**排练样本**：从 $\hat{A}_\text{old}$ 中采样 $(v, i)$

$$\text{target}_\text{reh}[v, i, :] = \hat{A}_\text{old}[v, i, :]$$

**注意**：$\hat{A}_\text{old}[v, i, :]$ 可从 CP 因子直接计算（$O(d \cdot D)$），无需展开 $K \times K \times D$ 全张量。

### 6.4 置信度掩码排练

标准排练对 $A_\text{old}$ 中未知位置（值接近零）施加 $\text{MSE}(A_\text{new}, 0)$，会将 CP 分解产生的微弱泛化信号压回零——"无知蒸馏"效应。

**解法**：只对 $\text{conf}(v,i) > \theta_\text{conf}$ 的位置执行排练：

$$\text{conf}(v,i) = \max_m |\hat{A}_\text{old}[v,i,m]|$$

未知位置被跳过，释放 CP 的泛化自由度。

### 6.5 完整 Sleep 优化目标

$$\mathcal{L} = \underbrace{\sum_{(v,i) \in \mathcal{B}} \lVert \hat{A}_\text{new}[v,i,:] - (\hat{A}_\text{old}[v,i,:] + B[v,i,:]) \rVert^2}_{\text{吸收新惊奇}} + \underbrace{\sum_{\substack{(v,i) \in \mathcal{R} \\ \text{conf}(v,i) > \theta_\text{conf}}} \lVert \hat{A}_\text{new}[v,i,:] - \hat{A}_\text{old}[v,i,:] \rVert^2}_{\text{置信度掩码排练}} + \underbrace{\lambda_\text{ortho} \lVert E^\top E - I \rVert_F^2}_{\text{正交锚定}}$$

### 6.6 排练样本采样

- **邻域排练**（50%）：B 中活跃 token 涉及的旧连接
- **随机补充**（50%）：全局正则化
- 排练样本总数 $\approx$ 新知识样本数（rehearsal_ratio = 1.0）

### 6.7 Sleep 流程

```
Sleep:
    active_set = {(v,i) : ‖B[v,i,:]‖ > ε}
    rehearsal_set = sample_neighbors(active_set) ∪ sample_random()

    if A is None:
        A = CP_fit(B, ortho_penalty=λ_ortho)           // 首次: 从头拟合
    else:
        A = finetune(A, B, active_set, rehearsal_set,
                     conf_mask=θ_conf,
                     ortho_penalty=λ_ortho)              // 后续: 增量微调

    B = 0    // 清零 B，释放容量
```

### 6.8 推荐参数

| 参数 | 推荐值 | 作用 |
|------|--------|------|
| $\lambda_\text{ortho}$ | 0.3 | 防旋转漂移 |
| $\theta_\text{conf}$ | 0.01 | 释放泛化自由度 |
| $d$ | 384（代码默认）；128–512（可调） | 语言有效秩（与 K 无关） |
| rehearsal_ratio | 1.0 | 排练/新知识样本比 |
| Sleep 优化器 | Adam, lr=0.01 | |
| Sleep epochs | 最大 10000，最小 500，patience=200 | |

## 7. 组件七：推理合并与解码

### 7.1 logit 合并

$$l[v] = l_B[v] + l_A[v]$$

两个通道的 logit 直接相加。B 提供精确的近期记忆，A 提供泛化的长期知识。

### 7.2 LayerNorm + Softmax

$$\hat{l}[v] = \gamma_\text{ln} \cdot \frac{l[v] - \mu_l}{\sigma_l + \epsilon}$$

$$p[v] = \frac{e^{\hat{l}[v]}}{\sum_{v'} e^{\hat{l}[v']}}$$

$\gamma_\text{ln} = 3 \ln V$ 使 softmax 在不同 $V$ 规模下保持适当的锐度。

### 7.3 自动加权机制

当 A 有信号而 B 无信号（新会话）：$l \approx l_A$，完全由长期知识驱动。

当 B 有强信号覆盖 A（会话中学到新事实）：B 的能量校准可在数次曝光内修正 A 的预测偏差。

当 A 和 B 一致：信号叠加，预测更确信。

这种加法合并不需要学习权重——B 的能量守恒写入天然与 A 的偏差成正比（$g = 1 - s_A - s_B$），实现了自平衡。

## 8. 组件八：Wake 完整单步

### 8.1 流程（严格时序）

给定当前位置的 token $t_\text{cur}$，已知 next-token $v^*$（训练时）或未知（推理时）：

```
1. Q 更新
   Q ← ρ ⊙ Q                    // 全局衰减
   Q[t_cur, :] += 1              // write=1

2. 距离特征提取
   active = {i : ‖Q[i,:]‖ > θ_Q}
   for i in active:
       φ[i] = L⁻¹ · Q[i,:]      // 固定线性投影

3. 预测
   φ̂[i] = φ[i] / ‖φ[i]‖          // 归一化 (B 通道)
   l_B = Σ_i B[:, i, :] · φ̂[i]   // B 通道 (归一化 φ)
   l_A = A_predict(active, φ̂)     // A 通道 (CP, 归一化 φ)
   l = l_B + l_A
   p = softmax(LayerNorm(l))

4. 输出
   v_out = argmax(p)              // 推理时
   // 或使用 v* = 实际 next-token  // 训练时

5. B 写入 (仅训练时, 统一突触能量守恒)
   B ← decay(B, λ_B)              // 惰性稀疏衰减（仅访问条目）
   φ̂[i] = φ[i] / ‖φ[i]‖          // 归一化
   for i in active:
       s_B = φ̂[i] · B[v*, i, :]   // B 投影强度
       s_A = φ̂[i] · Â[v*, i, :]   // A 投影强度 (CP 重构)
       g = 1 - (s_A + s_B)         // 双向能量缺口
       B[v*, i, :] += η · g · φ̂[i]
   prune(B, ε_prune)
```

### 8.2 时序说明

Q 更新在预测**之前**——当前 token 参与对 next-token 的预测。B 写入在预测**之后**——基于预测误差学习。这确保了因果性：不使用未来信息。

## 9. 组件九：读写模式

### 9.1 读写一致性

系统只有一种前向模式：读入 token → 更新 Q → 预测 next → 写入 B。训练和推理使用完全相同的前向计算，区别仅在于：

| | 训练（阅读模式） | 推理（输出模式） |
|---|---|---|
| next-token 来源 | 文档中的真实 token | argmax(p) |
| B 写入 | 有（能量守恒） | 有（从自身预测学习） |
| 序列终止 | 文档结束 | 生成 EOS 或达到长度限制 |

### 9.2 阅读模式

逐 token 读入文档，每步执行完整的 Wake 流程。B 持续积累预测误差驱动的连接更新。文档结束时 Q 重置（$Q \leftarrow 0$），但 **B 不清零**——跨文档的会话记忆保留在 B 中，直到下次 Sleep。

### 9.3 输出模式

从当前 Q 状态出发，循环执行 Wake 步骤。每步输出 $v_\text{out} = \text{argmax}(p)$，将 $v_\text{out}$ 作为下一步的 $t_\text{cur}$。B 仍然执行写入——系统从自身生成的序列中学习，这为后续推理提供即时上下文。

### 9.4 序列边界

不同文档之间：Q 重置，B 保留。

Sleep 触发时：B 巩固到 A，B 清零，Q 重置。

新会话开始：Q = 0，B = 0，仅靠 A 提供知识。

## 10. 组件十：并行训练

### 10.1 设计动机

单 Worker 串行处理文档太慢。并行训练让 $W$ 个 Worker 独立读取不同文档，各自积累 B，然后在 Sleep 时合并。

### 10.2 B 可平均的理论基础

能量守恒写入具有线性可加性：

$$B_\text{avg} = \frac{1}{W} \sum_{w=1}^{W} B_w$$

每个 $B_w$ 是同一分布数据的独立样本的能量校准累积。平均后 $B_\text{avg}$ 的期望等于全序列单 Worker 的 B，方差降低 $1/W$。

### 10.3 并行协议

```
并行 Wake:
    for each worker w = 1..W:
        B_w = 0
        for doc in worker_w_docs:
            read_document(doc, B_w)    // 各 Worker 独立累积 B_w

合并:
    B_merged = (1/W) · Σ_w B_w

Sleep:
    A = sleep(A, B_merged)
    B_merged = 0
```

### 10.4 通信开销

每轮 Sleep 前传输 $W$ 个 $K \times K \times D$ 张量（稀疏格式下远小于稠密大小）。Sleep 本身是单点执行。

## 11. 组件十一：稀疏化

### 11.1 设计原则

稀疏化不是外加的优化技巧，而是架构在大规模下的自然行为。三层自然稀疏机制从推理到训练逐层生效。

### 11.2 层 1：Q 的自然稀疏（推理时）

EMA 衰减使远距离 token 的 Q 值指数衰减至阈值以下。活跃 token 集 $K = |\{i : \|Q[i,:]\| > \theta_Q\}|$ 通常为几十，远小于 $V$。

推理时间从 $O(V^2 D)$ 降至 $O(K \cdot V \cdot D)$。

### 11.3 层 2：B 的自然竞争稀疏化

A 知识越丰富 → 大部分连接的 $s_A \approx 1$ → 能量缺口 $g \approx 0$ → B 的写入自然稀疏。这是能量守恒的内禀性质：当 A 已充分覆盖某连接时，B 无需写入。

实验验证（附录 H）：A 背景知识强时，B 只对真正的惊奇事件写入 1-2 个关键条目；A 知识弱时，B 对每个 token 位置都写入大量条目。

### 11.4 层 3：B 的稀疏存储

稠密 `np.zeros((V, V, D))` 在大 V 下不可行。切换为字典结构：

```python
B = defaultdict(dict)    # B[i][v] = D-vector, 只存非零条目
```

$\lambda_B$ 持续衰减 + $\epsilon_\text{prune}$ 裁剪确保非零条目数始终远小于 $V^2$。

### 11.5 综合效果

| 层 | 机制 | 推理节省 | 训练节省 |
|---|---|---|---|
| Q 稀疏 | EMA 衰减 | $V \to K$（source 维） | 同左 |
| B 自然稀疏 | 能量守恒 + A 知识 | 非零条目少 | 写入少 |
| B 稀疏存储 | 字典结构 | 内存 $O(K \cdot K')$ | 同左 |

## 12. 已知局限

### 12.1 B 写入的因果归因瓶颈

B 的写入以 Q 为加权，对所有活跃 source 的 $(v^*, i)$ 连接执行能量校准。它记录的是"谁在场时 $v^*$ 的连接需要补偿"，而非"谁导致了 $v^*$ 的出现"。在高度模糊的语境中（多个 source 同时活跃），归因精度受限。

### 12.2 CP 分解的非确定性

三阶张量的 CP 分解是 NP-Hard 问题，依赖梯度下降，存在局部最优。伪排练 + 正交约束 + warm start 在实践中缓解了此问题，但理论上不保证全局最优。

### 12.3 B 有效容量依赖 A 知识

附录 H 验证：A 知识贫乏时 B 只能记住 1-2 个新事实（每个文档全是惊奇），A 知识丰富时 B 可记住大量新事实（仅 delta 是惊奇）。这是理想的缩放性质，但意味着系统早期需要课程学习。

### 13.1 完整生命周期

```
初始化:
    Q = 0, B = 0 (稀疏), A = None (无长期知识)
    L⁻¹ = precompute(ρ, τ, λ_reg)

训练循环:
    for each curriculum_stage:
        data = stage_data ∪ review_data    // 渐进交错
        for doc in data:
            Q.reset()
            for token in doc:
                wake_step(token)            // §7 完整流程

        if sleep_trigger():                 // B 范数/密度阈值
            A = sleep(A, B)                 // §5 伪排练巩固

部署后:
    Q = 0, B = 0
    for token in input_stream:
        wake_step(token)                    // A 提供背景知识
                                            // B 积累会话记忆
    // 可选: 周期性 Sleep 巩固新知识到 A
```

### 13.2 渐进交错训练协议

新知识引入后与旧知识混合训练（附录 G）：

```
阶段 1: 数据₁ × N₁
阶段 2: (数据₁ + 数据₂) × N₂
阶段 3: (数据₁ + 数据₂ + 数据₃) × N₃
...
```

实验验证：渐进交错用更少的 Sleep 达到 100% 训练保持，优于纯顺序训练（72%）和等量 Sleep 的间隔复习（87%）。

### 13.3 课程学习框架

| 阶段 | 数据 | 目标 | Q 尺度 |
|------|------|------|--------|
| 1: 语法骨架 | 高频对话 | A 学会高频转移模式 | $k=0$ |
| 2: 词汇语义 | 词典 + 短句 | 建立嵌入差异 | $k=0,1$ |
| 3: 短文阅读 | 儿童故事 | 跨句推理 | $k=1,2$ |
| 4: 复杂文档 | 百科/教科书 | 专业知识 | $k=2,3$ |

课程学习是架构的**数学必要条件**：A 必须先建立基础结构，后续 B 的惊奇足迹才能缩小，学习效率才能提升。

- **固定间隔**：每 $N$ 个文档后触发
- **自适应触发**：监控 $\|B\|_F$ 或 B 的非零条目密度，超过阈值时触发

## 14. 运算复杂度总结

### 14.1 单步复杂度（稠密）

| 操作 | 复杂度 | 备注 |
|------|--------|------|
| Q 更新 | $O(V \cdot N)$ | 全局衰减 + 单点写入 |
| $\varphi$ 计算 | $O(K \cdot N \cdot D)$ | $K$ 个活跃 token |
| B 预测 | $O(K \cdot V \cdot D)$ | |
| A 预测 | $O(K \cdot d \cdot D + V \cdot d)$ | CP 结构 |
| LayerNorm + softmax | $O(V)$ | |
| B 写入 | $O(K_\text{target} \cdot K \cdot D)$ | $K_\text{target}$: target 的活跃槽位数 |
| **单步总计** | $O(K \cdot V \cdot D)$ | B 预测主导 |

### 14.2 单步复杂度（稀疏化后）

B 稀疏存储下，B 预测只访问非零条目：$O(K \cdot \text{nnz}_\text{per\_source} \cdot D)$。

A 知识丰富后，能量守恒使大部分连接 $g \approx 0$，B 写入量极小，接近 $O(K \cdot D)$。

### 14.3 Sleep 复杂度

| 操作 | 复杂度 |
|------|--------|
| CP 因子微调 | $O(\text{epochs} \cdot (n_\text{active} + n_\text{reh}) \cdot d \cdot D)$ |
| 正交约束 | $O(V \cdot d^2)$（计算 $E^\top E$） |

### 14.4 存储复杂度

| 组件 | 大小 | 备注 |
|------|------|------|
| Q | $K \times N$ | 会话状态 |
| $E_\text{proj}$ | $D \times N$ | 固定，极小 |
| B（稀疏） | $\ll K^2 \times D$ | 非零条目数随 A 知识增长而减少 |
| A（CP 因子） | $K \cdot d + 2d^2 + D \cdot d$ | 长期知识，$d$ = 384 |

## 附录 A：超参数自适应推导

### A.1 $\rho_k$ 的选择

窗口宽度 $w_k$ 对数等分于 $[w_\min, w_\max]$：

$$w_k = w_\min \cdot \left(\frac{w_\max}{w_\min}\right)^{k/(N-1)}$$

$\rho_k = 1 - 1/w_k$ 自动满足 $0 < \rho_0 < \cdots < \rho_{N-1} < 1$。

推荐 $w_\min = 2$（紧邻 bigram）、$w_\max = 100$（段落级）、$N = 4$。

### A.2 $\gamma_\text{ln}$ 的推导

LayerNorm 后 softmax 的锐度由 $\gamma_\text{ln}$ 控制。令 top-1 与 top-2 的 logit 差为 $\Delta$，LayerNorm 后差为 $\gamma \cdot \Delta / \sigma_l$。要求 softmax 输出的 top-1 概率 $> 0.5$，推导出：

$$\gamma_\text{ln} \approx 3 \ln V$$

在 $V=100$ 到 $V=100000$ 范围内均给出合理的 softmax 锐度。

### A.3 $\eta$ 的选择

能量守恒写入的学习率 $\eta$ 控制 B 的单步写入幅度。$\eta = 0.5$ 时，压缩映射 $S_\text{new} = (1-\eta)S + \eta$ 以每步 50% 的速率趋近不动点 1，兼顾收敛速度与稳定性。

推荐 $\eta = 0.5$。能量守恒的自然制动（$g \to 0 \Rightarrow \Delta B \to 0$）使系统对 $\eta$ 的敏感度较低。

### A.4 $\lambda_B$ 的选择

全局衰减率控制 B 中旧信息的自然遗忘速度。$\lambda_B = 1 \times 10^{-4}$ 对应约 7000 步的半衰期，在典型文档长度下足够。

### A.5 $d$ 的选择

CP 分解的秩 $d$ 控制 A 的容量-泛化平衡。$d$ 的本质是连接模式的**有效秩**——语言中独立的语法/语义角色总数是有限的，不随词表 $V$ 线性增长。

**小词表实验**（V=119）中 $d \approx V/2 \approx 60$ 表现最优，但这是因为 60 恰好匹配了该词表的语言复杂度，不是因为 $d$ 应与 $V$ 成正比。

**真实语料的稀疏性证据**：WikiText-103 实测显示，每个 token 的有效连接数（非零 B 条目）基本稳定在几十到几百，不随 V 增长。这意味着 B 的 source 维度上的有效秩是常数级的——如果将 $d$ 视为嵌入空间中的"虚拟词表"，真正需要区分的角色数远小于 $V$。

**推荐**：

| K 规模 | 推荐 $d$ | 依据 |
|--------|---------|------|
| K ≤ 200（合成实验） | $\approx K/2$ | 实验验证 |
| K = 65536（真实系统） | **384（代码默认）** | 语言有效秩的上界 |

$d$ 不应随 $K$ 线性扩展——这是 CP 分解的核心压缩优势。存储从 $K^2 D$ 压缩到 $Kd$，当 $d$ 为常数时压缩比随 $K$ 线性增长。

### A.6 距离中心 $\tau$ 的选择

近端密、远端疏（Weber 定律）：$\tau_m = \text{round}(d_\max \cdot (m/(D-1))^{1.5})$。

$D=10$, $d_\max = 20$ 给出近端 5 个点（$\tau = 0,1,2,4,6$）、远端 5 个点（$\tau = 8,11,14,17,20$）。

## 附录 B：关键数学性质汇总

### B.1 EMA 编码的唯一性

对固定 $\rho \in (0,1)$，$\sum_m \rho^{\delta_m}$ 对不同的 $\{\delta_m\}$ 产生不同的值。$N$ 个通道提供数值冗余。

### B.2 逆 Laplace 投影的稳定性

$L^{-1} = M^\top(MM^\top + \lambda I)^{-1}$。Tikhonov 正则化 $\lambda > 0$ 确保条件数有限。$N=4$ 优于 $N=6/8$（更多通道导致 Vandermonde 矩阵条件数恶化）。

### B.3 能量守恒的压缩映射

令 $S = s_A + s_B$ 为总投影强度，单步更新后 $S_\text{new} = (1-\eta)S + \eta$。这是以 1 为不动点的压缩映射，无论初值如何都保证收敛，不可能振荡。

### B.4 能量守恒的自然制动

$S \to 1 \Rightarrow g = 1 - S \to 0 \Rightarrow \Delta B \to 0$。已学好的连接自动停止写入，无需外部饱和函数。双向写入（$g$ 可负）进一步保证 A 的 CP 过冲也能被修正。

### B.5 正交约束的相变行为

$\lambda_\text{ortho} \geq 0.01$ 即可将 $\lVert E^\top E - I \rVert_F$ 从 $O(10)$ 降至 $O(10^{-4})$。这是优化景观的几何性质——正交状态是自然稳定点。

### B.6 CP 投影空间的泛化

$E \cdot W_Q$ 和 $E \cdot W_V$ 的矩阵乘法从嵌入空间提取语义共性。$\cos(E[\text{fox}], E[\text{cat}]) \approx 0.05$ 但 $\cos(\hat{A}[\text{fox, eat}], \hat{A}[\text{cat, eat}]) \approx 0.70$。

### B.7 B 有效容量公式

$$\text{B 有效容量} \approx \frac{V \times V \times D}{\text{每事实平均惊奇条目数}}$$

A 知识增长自然减少惊奇足迹，等价于自动稀疏化。

### B.8 连接稀疏性与 $d$ 的常数性

真实语料中每个槽位的有效连接数不随 $K$ 线性增长，而是基本稳定。这意味着 B 的 source 维度的有效秩是常数级的，$d$ 不需要随 $K$ 扩展。$d$ = 384（代码默认）足以覆盖语言中独立语法/语义角色的总数。

## 附录 C：理解本架构的四个关键视角

### C.1 范畴化前提

离散 token 作为系统的基本单元，不仅是计算便利，而且与生物神经系统的范畴化响应存在结构性对应。大脑处理的也不是连续原始信号，而是经过范畴化的离散特征成分。

### C.2 仿生视角：启发与佐证

| 设计选择 | 生物对应 |
|---------|---------|
| B 的能量守恒写入 | 赫布学习 + 突触容量约束 |
| Wake-Sleep 双阶段 | 互补学习系统（CLS） |
| $\lambda_B$ 稀疏衰减 | 突触自然退化 |
| $E_\text{proj}$ 固定投影 | 前馈兴奋-抑制连接 |
| Q 多尺度 EMA | Laplace cells |
| $\varphi$ 距离特征 | Timing cells |

仿生是源于生物但高于生物的过程——提取核心计算原理并用数学精确表达，最终判据是数学自洽性和实验效果。

### C.3 表达能力：语境匹配

$l_B[v] = \sum_{i,m} B[v,i,m] \cdot \varphi[i,m]$ 不是 token-to-token 的二元查表。$\varphi$ 编码了完整语境的多尺度距离指纹，B 的点积对整个语境模式做响应，天然包含多 token 联合效应。

### C.4 动态连续推理

单步计算简单（线性点积 + LayerNorm + softmax），但系统的行为复杂度来自多步 argmax 的链式传播。每步 argmax 是极端非线性操作——将概率分布坍缩为离散选择。多步 argmax 的累积效应产生的行为复杂度远超单步线性所暗示的。"深度"不在空间上的层数，而在时间上的推理步数。

## 附录 E：Cosine 混沌编码 — 替代 Laplace 逆变换的距离特征提取

### E.1 问题：Laplace 逆变换的局部区分度瓶颈

原始设计使用 Laplace 逆变换 $\varphi = L^{-1} \cdot Q$ 将 EMA 编码的 Q 转换为距离特征向量。实验发现该方法存在根本性缺陷：

**现象**：在长上下文（100+ tokens）中，相邻位置的 $\varphi$ 高度相似。

| 上下文长度 | 相邻位置 $\varphi$ 余弦相似度 |
|-----------|---------------------------|
| 13 tokens | 0.87 - 0.99 |
| 100 tokens | 0.96 - 0.99 |

**根本原因**：

1. **EMA 的累积效应**：长上下文中 Q 值被历史累积主导，新 token 的增量 (+1) 相对于累积值占比极小
2. **线性投影的局限**：$L^{-1}$ 是固定线性变换，无法放大微小的 Q 差异
3. **架构矛盾**：MathBrain 依赖"连续局部模式匹配"，但 Laplace 逆变换的局部区分度太弱，长程区分度反而更好——与架构期望的运行模式矛盾

**后果**：

- 不同局部模式（如 "into the" vs "out of"）的 $\varphi$ 差异被淹没
- B 写入时无法有效区分不同上下文
- 高频词的 B 范数累积淹没低频词
- 模型输出退化为重复高频词

### E.2 解决方案：Cosine 混沌编码

引入非线性变换替代 Laplace 逆变换：

$$\varphi = \cos(\alpha \cdot E_{\text{proj}} \cdot Q)$$

其中：
- $E_{\text{proj}} \in \mathbb{R}^{D \times N}$：随机投影矩阵（初始化后固定）
- $\alpha$：混沌强度参数（推荐值 3.0-5.0）
- $\cos$：逐元素余弦函数

**设计理由**：

1. **空间折叠**：$\cos$ 的周期性（周期 $2\pi$）将连续空间折叠，微小的输入差异可能导致输出跨越周期边界
2. **范数敏感**：折叠是否触发取决于 $\alpha \cdot E_{\text{proj}} \cdot Q$ 的范数大小
3. **伪随机而非真混沌**：不同上下文的 $\varphi$ 不是严格正交，而是"伪随机"分布，保留泛化可能性

### E.3 核心数学猜想：EMA + Cosine 混沌的协同机制

**猜想**：EMA 衰减与 Cosine 混沌形成协同效应，自动实现"近邻高区分度 + 远程高泛化度"的理想特性。

**机制分析**：

1. **EMA 提供范数梯度**：
   - 距离 $\delta$ 步的 token 对 Q 的贡献为 $\rho^\delta$
   - 远处 token（$\delta$ 大）：$\rho^\delta \to 0$，贡献的范数小
   - 近处 token（$\delta$ 小）：$\rho^\delta \approx 1$，贡献的范数大

2. **Cosine 将范数梯度转化为混沌梯度**：
   - 范数小 → $\alpha \cdot E_{\text{proj}} \cdot \Delta Q$ 小 → 不触发 $\cos$ 折叠 → 输出变化平滑 → 相似度高
   - 范数大 → $\alpha \cdot E_{\text{proj}} \cdot \Delta Q$ 大 → 触发 $\cos$ 折叠 → 输出变化剧烈 → 相似度低

3. **结果**：
   - 远处 token 变化 → $\varphi$ 相似度高 → 泛化
   - 近处 token 变化 → $\varphi$ 相似度低 → 区分

**数学表述**：

设上下文长度为 $L$，改变位置 $j$ 的 token，则 $\varphi$ 的变化程度 $\Delta\varphi$ 满足：

$$\mathbb{E}[\|\Delta\varphi\|] \propto f(\alpha \cdot \|\rho^{L-1-j}\|)$$

其中 $f$ 是与 $\cos$ 折叠相关的非线性函数。当 $\rho^{L-1-j}$ 小（远处 token）时，$\Delta\varphi$ 小；当 $\rho^{L-1-j}$ 大（近处 token）时，$\Delta\varphi$ 大。

### E.4 实验验证

#### E.4.1 位置与混沌度的关系

测试：改变不同位置的 token，观察 $\varphi$ 相似度变化。

| 上下文长度 | 位置 0 (最早) $\varphi$ 相似度 | 位置 L-1 (最近) $\varphi$ 相似度 |
|-----------|------------------------------|--------------------------------|
| 50 | 0.33 | 0.18 |
| 100 | 0.66 | 0.10 |
| 200 | 0.93 | 0.11 |
| 500 | 1.00 | 0.00 |
| 1000 | 1.00 | -0.01 |

**结论**：距离越远的 token，改变后 $\varphi$ 相似度越高（混沌度越低），验证了猜想。

#### E.4.2 衰减率与泛化能力的关系

测试：不同 EMA 衰减率下的局部模式泛化能力。

场景：训练 5000 个 "早期上下文 + into the → room" 样本，测试新的早期上下文能否正确预测。

| 衰减率配置 | 泛化准确率 | B[room]·B[house] 相似度 |
|-----------|-----------|------------------------|
| 原版 [0.5, 0.8, 0.95, 0.99] | 48% | 0.97 |
| 快速 [0.3, 0.5, 0.7, 0.85] | 55% | 0.64 |
| 极快 [0.2, 0.4, 0.6, 0.75] | 85% | 0.28 |
| 超快 [0.1, 0.3, 0.5, 0.7] | **92%** | 0.10 |

**结论**：更快的衰减率让局部模式主导 $\varphi$，不同局部模式的 B 向量方向分离，泛化准确率提升。

#### E.4.3 B 向量方向的收敛行为

测试：大量训练后 B 向量方向的变化。

| 训练样本数 | B[room]·B[house] 相似度 (原版衰减) |
|-----------|----------------------------------|
| 100 | 0.39 |
| 1000 | 0.75 |
| 10000 | 0.81 |

**发现**：原版衰减率下，大量训练后不同目标的 B 向量方向趋同（相似度增加），导致无法区分。快速衰减率避免了这一问题。

### E.5 与 MathBrain 架构的契合

MathBrain 的记忆核心思想是"局部模式匹配 → 幻想记忆"，而非精确检索。这要求：

1. **局部模式高区分度**：不同的近邻模式（如 "into the" vs "out of"）必须产生不同的 $\varphi$
2. **远程语境可泛化**：不同的早期上下文 + 相同的局部模式应产生相似的 $\varphi$
3. **抗高频淹没**：高频词不应因累积范数大而淹没低频词的局部模式

Cosine 混沌 + 快速 EMA 衰减完美满足这三个要求：

| 需求 | Laplace 逆变换 | Cosine 混沌 + 快速衰减 |
|------|---------------|----------------------|
| 局部区分度 | 弱（相似度 0.87-0.99） | 强（相似度 -0.69） |
| 远程泛化 | 差（长程反而区分开） | 好（相似度 → 1.0） |
| 抗高频淹没 | 差（线性累积） | 好（非线性折叠） |

### E.6 记忆构建的新范式

基于 Cosine 混沌编码，记忆构建变为：

$$\text{记忆} = \underbrace{\text{长尾 Q（统计语境）}}_{\text{低混沌，提供泛化}} + \underbrace{\text{近期 Q（局部模式）}}_{\text{高混沌，提供区分}}$$

- **长尾 Q**：从统计意义上提供语境背景，由于衰减严重不触发混沌，不干扰局部匹配
- **近期 Q**：提供丰富的检索/匹配条件，高混沌度确保不同局部模式可区分
- **整体特性**：高连续性，无明显阶梯，符合自然语言的渐进语义结构

### E.7 推荐配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| $\alpha$ | 3.0 - 5.0 | 混沌强度，过大导致过度混沌，过小区分度不足 |
| $E_{\text{proj}}$ | $\mathcal{N}(0, 0.25) \in \mathbb{R}^{D \times N}$ | 随机投影矩阵，初始化后固定 |
| $\rho$ | [0.2, 0.4, 0.6, 0.75] 或更快 | 快速衰减让局部模式主导 |

### E.8 多尺度 Cosine 混沌

#### E.8.1 动机

为不同时间尺度 $k$ 设置不同的 $\alpha_k$，实现更精细的混沌控制。类似于大脑的网格细胞（Grid Cells），通过不同空间频率的激活模式来编码位置信息。

#### E.8.2 多尺度 $\alpha$ 的设计

**公式**：

$$\varphi = \cos(E_{\text{proj}} \cdot (\boldsymbol{\alpha} \odot Q))$$

其中 $\boldsymbol{\alpha} = [\alpha_0, \alpha_1, \ldots, \alpha_{N-1}]$，$\odot$ 表示逐元素乘法。

**两种设计思路**：

| 思路 | 公式 | 理由 |
|------|------|------|
| 正向缩放 | $\alpha_k \propto 1/(1-\rho_k)$ | 让各尺度的混沌触发阈值相当 |
| 反向缩放 | $\alpha_k \propto (1-\rho_k)$ | 让快尺度高混沌、慢尺度低混沌 |

#### E.8.3 实验对比

测试配置（$\rho = [0.3, 0.6, 0.85, 0.95]$）：

| 配置 | $\alpha_0$ (快) | $\alpha_1$ | $\alpha_2$ | $\alpha_3$ (慢) |
|------|----------------|------------|------------|-----------------|
| 统一 | 5.00 | 5.00 | 5.00 | 5.00 |
| 正向缩放 | 0.93 | 1.63 | 4.36 | 13.07 |
| 反向缩放 | 10.77 | 6.15 | 2.31 | 0.77 |

**泛化测试结果**（3000 训练样本，200 测试样本）：

| 配置 | 泛化准确率 | B[room]·B[house] 相似度 |
|------|-----------|------------------------|
| 统一 $\alpha=5$ | 71% | 0.999 |
| 正向缩放 | 7% | 0.955 |
| **反向缩放** | **95%** | 0.830 |

#### E.8.4 核心发现

**反向缩放 $\alpha_k \propto (1-\rho_k)$ 效果最好**，这与直觉相反但符合 MathBrain 的需求：

| 尺度 | $\rho_k$ | $\alpha_k$ (反向) | 作用 |
|------|----------|-------------------|------|
| 0 (快) | 0.30 | 10.77 | 高混沌 → 近邻精确区分 |
| 1 | 0.60 | 6.15 | 中等混沌 |
| 2 | 0.85 | 2.31 | 低混沌 |
| 3 (慢) | 0.95 | 0.77 | 极低混沌 → 远程语境泛化 |

**原理解释**：

- **正向缩放的问题**：慢尺度 $\alpha$ 太大（13.07），导致远程语境也产生高混沌，破坏了泛化能力
- **反向缩放的优势**：
  - 快尺度（近邻）：$\Delta Q_k$ 大 + 大 $\alpha$ → 高混沌 → 精确区分局部模式
  - 慢尺度（远程）：$\Delta Q_k$ 小 + 小 $\alpha$ → 低混沌 → 宏观语境泛化

#### E.8.5 与 Cosine 混沌对 Sleep 的正面影响

实验验证 Cosine 混沌编码对 CP 分解的影响：

| 指标 | Laplace 逆变换 | Cosine 混沌 |
|------|---------------|-------------|
| $\varphi$ 范数分布 | mean=9.94, std=4.07 | mean=2.00, std=0.25 |
| 协方差矩阵有效秩 | 1.94 (退化) | **7.95** (接近满秩) |
| 协方差矩阵条件数 | $3 \times 10^{17}$ (病态) | **1.5** (良好) |
| $\varphi$ 相似度 | mean=0.27 | mean=-0.01 (正交) |

**结论**：Cosine 混沌将 $\varphi$ 均匀散布在超球面上，产生高度正交、满秩的特征，极大改善 Sleep 阶段 CP 分解的收敛性和稳定性。

#### E.8.6 多尺度 Cosine 混沌的推荐配置

$$\alpha_k = \frac{(1-\rho_k)}{\text{mean}(1-\rho)} \cdot \alpha_{\text{base}}$$

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| $\alpha_{\text{base}}$ | 5.0 | 基础混沌强度 |
| 缩放方式 | 反向 $\propto (1-\rho_k)$ | 快尺度大、慢尺度小 |

### E.9 理论分析

#### E.9.1 反向缩放是"能量归一化"算子

**稳态能量分析**：持续输入常数信号时，EMA 缓冲的稳态值为：

$$Q_{k, \infty} = \frac{1}{1-\rho_k}$$

不同衰减率的稳态能量差异巨大（$\rho=0.3$ 时 $Q_\infty=1.43$，$\rho=0.95$ 时 $Q_\infty=20$）。

**能量归一化验证**：

| 配置 | $\alpha_k \cdot Q_{k,\infty}$ 的标准差 |
|------|--------------------------------------|
| 统一 $\alpha$ | 36.97 |
| 正向缩放 | 108.79 |
| **反向缩放** | **0.00** |

反向缩放 $\alpha_k \propto (1-\rho_k)$ 完美抵消了稳态能量差异，确保各尺度的相空间体积被严格钳制在同一数量级。

#### E.9.2 $\alpha_{\text{base}}$ 的自适应公式

**理论推导**：希望新 Token（$\Delta Q = 1$）产生约 $\pi/2$ 的相位变化以保证正交性：

$$\mathbb{E}[\|\alpha_{\text{base}} \cdot E_{\text{proj}} \cdot \Delta Q\|] \approx \frac{\pi}{2}$$

由于 $E_{\text{proj}} \sim \mathcal{N}(0, \sigma^2)$，其范数期望 $\approx \sigma\sqrt{D}$，因此：

$$\boxed{\alpha_{\text{base}} = \frac{\pi}{2\sigma\sqrt{D}}}$$

**实验验证**（$\sigma=0.5$, $D=8$）：

| $\alpha$ | $\Delta Q=1$ 时的相位变化 |
|----------|-------------------------|
| 1.11 (理论值) | 1.17 ≈ $\pi/2$ ✓ |
| 5.00 | 5.25 |

#### E.9.3 与随机傅里叶特征（RFF）的联系

Cosine 混沌编码 $\cos(\alpha \cdot E_{\text{proj}} \cdot Q)$ 在数学上类似于 Rahimi & Recht (2007) 提出的随机傅里叶特征，用于逼近高斯核。

**关键性质**：

1. **范数到相位的降维打击**：无论 Q 的历史累积多大，$\cos$ 将其折叠到 $[-1, 1]$ 的有界空间，消除高频词累积范数淹没问题

2. **导数放大器**：根据链式法则，$\frac{d\varphi}{dQ} \propto -\sin(\ldots) \cdot \alpha$
   - 近期 token（$\Delta Q$ 大）：变化被放大 → 高区分度
   - 远期 token（$\Delta Q$ 小）：变化被抑制 → 高泛化度

#### E.9.4 语义饱和特性（vs RoPE）

**对比**：

| 特性 | RoPE (Transformer) | Cosine 混沌 (MathBrain) |
|------|-------------------|------------------------|
| 编码基础 | 绝对物理时间 $t$ | 语义积累（EMA 范数） |
| 重复输入 | 相位持续旋转 | **相位饱和停止** |
| 信息过滤 | 无 | 自动过滤冗余信息 |

**实验验证**（反向缩放 $\alpha$）：

| 场景 | $\varphi$ 相似度 |
|------|-----------------|
| 重复输入相同 Token（与上一步） | **1.0000**（饱和） |
| 稳态后输入相同 Token | 0.9996 |
| 稳态后输入不同 Token | -0.5452（仍能区分） |

**结论**：Cosine 混沌是"语义 RoPE"——对重复信息饱和，对新信息敏感。

#### E.9.5 李雅普诺夫指数特性

EMA 的差分 $\Delta Q$ 按 $\rho^t$ 指数衰减，代入 Cosine 混沌编码后：

$$\|\Delta\varphi\| \leq C \cdot \alpha \cdot \rho^t$$

这严格证明了"时间越近，相空间分裂越快；时间越远，相空间越稳定"的动态系统特性。

#### E.9.6 网格细胞对应

多尺度 Cosine 混沌在数学上重现了哺乳动物内嗅皮层网格细胞的放电机制（2014 年诺贝尔生理学奖）：

| MathBrain 尺度 | $\rho_k$ | $\alpha_k$ | 相位变化率 | 网格细胞对应 |
|---------------|----------|------------|-----------|-------------|
| 0 (快) | 0.30 | 10.77 | 1.59 | 背侧（密集网格，精确定位） |
| 1 | 0.60 | 6.15 | 1.87 | 背侧 |
| 2 | 0.85 | 2.31 | 1.27 | 中间 |
| 3 (慢) | 0.95 | 0.77 | 0.40 | 腹侧（稀疏网格，宏观定位） |

### E.10 开放问题（已部分解决）

1. ✅ **$\alpha_{\text{base}}$ 的自适应选择**：$\alpha_{\text{base}} = \pi/(2\sigma\sqrt{D})$

2. **多尺度维度分配**：实验显示均匀分配已足够（100% 泛化），但理论上快尺度应分配更多维度（信息熵匹配）

3. ✅ **与 RoPE 的对比**：Cosine 混沌是"语义 RoPE"，基于语义积累而非物理时间，具有自动饱和特性

4. ✅ **生物对应**：多尺度混沌完美对应网格细胞的背侧/腹侧分布

## 附录 F：HashRetina — 哈希视网膜编码机制

### F.1 设计动机

MathBrain 的核心架构在 token ID 空间上运行，每个词对应一个独立的维度。这种设计在语义层面是正确的（cat 和 kitten 是不同的词），但在字形层面丢失了重要信息：

- **形态变化**：apple/apples、run/running 共享大部分字符结构
- **拼写相似**：cat/car 只差一个字符，但在独热编码中完全正交
- **未登录词**：训练时未见过的词无法获得任何先验信息

HashRetina（哈希视网膜）是一个**零参数、确定性**的前端编码器，将词的字符序列映射到稀疏的虚拟槽位空间。其核心思想是：

> **用 N-gram 哈希捕捉字形结构，让形态相似的词共享部分槽位，从而实现字形级别的泛化。**

### F.2 V1 架构：Bigram + 位置编码

V1 版本使用 bigram（字符二元组）结合微观 EMA 位置编码：

**步骤 1：字符级微观 EMA**

对词 $w$ 的字符序列 $c_1, c_2, \ldots, c_L$（加上边界符 `^` 和 `$`），维护一个 $N_\text{micro}$ 维的微观 EMA 状态：

$$Q_\text{micro} \leftarrow Q_\text{micro} \cdot \rho_\text{micro} + \text{hash}(c_i)$$

其中 $\rho_\text{micro} = [0.3, 0.6, 0.85]$（默认 $N_\text{micro}=3$）。

**步骤 2：位置特征提取**

通过逆 Laplace 投影提取位置特征：

$$\phi_\text{pos} = L^{-1}_\text{micro} \cdot Q_\text{micro}$$

然后量化为离散值：$\phi_\text{quantized} = \text{int}(\phi_\text{pos} \times 1000) \mod 10000$

**步骤 3：槽位哈希**

将 bigram 与量化位置特征组合后哈希到槽位：

$$\text{slot\_id} = \text{MD5}(\text{bigram} \oplus \phi_\text{quantized}) \mod K$$

**V1 特性**：
- 槽位数 $K = 2048$（默认）
- 同一 bigram 在不同位置映射到不同槽位（位置敏感）
- 返回槽位集合（无频次信息）

### F.3 V2 架构：N-gram 频次统计

V2 版本简化了编码流程，使用纯 N-gram 统计：

**步骤 1：N-gram 提取**

对词 $w$ 加边界符后提取所有 N-gram（默认 $n=3$，即 trigram）：

```
word = "apple"
padded = "^apple$"
trigrams = ["^ap", "app", "ppl", "ple", "le$"]
```

**步骤 2：频次统计**

统计每个 N-gram 的出现次数（处理重复 N-gram）：

$$\text{ngram\_counts}[\text{ng}] = \sum_{i} \mathbb{1}[\text{padded}[i:i+n] = \text{ng}]$$

**步骤 3：槽位映射**

将 N-gram 哈希到槽位并累加频次：

$$\text{slot\_counts}[j] = \sum_{\text{ng}: \text{hash}(\text{ng}) = j} \text{ngram\_counts}[\text{ng}]$$

**V2 特性**：
- 槽位数 $K = 65536$（更大以减少哈希冲突）
- 无位置编码（N-gram 本身蕴含顺序信息）
- 返回频次字典 `Dict[slot_id, count]`
- 频次作为 Q 更新时的刺激强度

### F.4 槽位共享与字形泛化

HashRetina 的核心价值在于**槽位共享**机制：

**形态变化共享**

| 词对 | 共享 trigrams | Jaccard 相似度 |
|------|---------------|----------------|
| apple / apples | `^ap`, `app`, `ppl`, `ple` | ~0.67 |
| cat / cats | `^ca`, `cat`, `at$` → `ats` | ~0.50 |
| run / running | `^ru`, `run` | ~0.33 |

**泛化机制**

当模型学习了 "the cat eat fish" 后：
- B 中记录了 `cat` 的槽位与 `eat` 的槽位之间的连接
- 遇到未见过的 `cats` 时，由于 `cats` 与 `cat` 共享约 50% 的槽位
- 这些共享槽位的 B 连接被部分激活，产生泛化预测

**Jaccard 相似度计算**

$$J(w_1, w_2) = \frac{|S_{w_1} \cap S_{w_2}|}{|S_{w_1} \cup S_{w_2}|}$$

其中 $S_w$ 是词 $w$ 的槽位集合。

### F.5 V2 的 Sigmoid 多标签分类

V2 架构使用 Sigmoid 多标签分类替代 Softmax：

**编码阶段**

目标词 $w_\text{target}$ 被编码为稀疏二值向量：

$$s_\text{target}[j] = \begin{cases} 1 & \text{if } j \in S_{w_\text{target}} \\ 0 & \text{otherwise} \end{cases}$$

**预测阶段**

对每个槽位独立预测：

$$\hat{y}[j] = \sigma(l[j]) = \frac{1}{1 + e^{-l[j]}}$$

**写入阶段**

误差信号：

$$e[j] = s_\text{target}[j] - \sigma(l[j])$$

- 正样本（$j \in S_{w_\text{target}}$）：$e[j] = 1 - \sigma(l[j])$，推高预测
- 负样本（$j \notin S_{w_\text{target}}$ 且 $\sigma(l[j]) > 0.1$）：$e[j] = -0.1 \times \sigma(l[j])$，压低预测（系数 0.1 避免过度抑制）

**与 Softmax 的区别**

| | Softmax | Sigmoid 多标签 |
|---|---------|----------------|
| 输出 | 概率分布（和为 1） | 独立概率（各自 [0,1]） |
| 竞争 | 全局竞争（提高一个必降低其他） | 无竞争（各槽位独立） |
| 适用 | 单标签分类 | 多标签分类（一词多槽位） |
| 梯度 | 分散到所有类 | 集中在相关槽位 |

### F.6 解码：从槽位分数到词分数

**V1/V2 解码（算术平均）**

$$\text{Score}_w = \frac{1}{|S_w|} \sum_{j \in S_w} l[j]$$

算术平均消除了长词偏差（长词有更多槽位，原始求和会偏高）。

### F.7 Q 更新的频次加权

V2 架构中，Q 更新时使用频次作为刺激强度：

```python
slot_counts = retina.encode(word)  # Dict[slot_id, count]
for slot_id, count in slot_counts.items():
    Q_sparse[slot_id] += float(count)  # 频次作为刺激强度
```

这意味着：
- 词中重复出现的 N-gram 对 Q 的贡献更大
- 长词的总刺激强度更高（但解码时被平均消除）

### F.8 超参数

| 参数 | V1 | V2 | 含义 |
|------|----|----|------|
| $K$ | 2048 | 65536 | 虚拟槽位数 |
| $n$ | 2 (bigram) | 3 (trigram) | N-gram 大小 |
| $N_\text{micro}$ | 3 | — | 微观 EMA 尺度数（V1 专用） |
| $\rho_\text{micro}$ | [0.3, 0.6, 0.85] | — | 微观 EMA 衰减率（V1 专用） |

**$K$ 的选择**

$K$ 应足够大以减少哈希冲突。对于词汇表大小 $V$，每个词平均 $\bar{n}$ 个 N-gram：

$$\text{冲突概率} \approx 1 - e^{-V \cdot \bar{n}^2 / (2K)}$$

$V = 10000$，$\bar{n} = 5$ 时，$K = 65536$ 的冲突概率约 16%。

**$n$ 的选择**

- $n = 2$ (bigram)：捕捉局部字符对，泛化能力强但区分度低
- $n = 3$ (trigram)：平衡泛化与区分，推荐默认值
- $n = 4$ (4-gram)：区分度高但泛化能力弱

### F.9 与主架构的集成

HashRetina 作为前端编码器，与 MathBrain 主架构的集成方式：

```
输入词 w
    │
    ▼
HashRetina.encode(w) → 槽位集合/频次字典
    │
    ▼
Q 更新（在槽位空间）
    │
    ▼
φ 计算（Laplace 或 Cosine Chaos）
    │
    ▼
B/A 预测（在槽位空间）
    │
    ▼
解码（槽位分数 → 词分数）
    │
    ▼
输出预测
```

**关键点**：
- Q、B、A 全部在槽位空间运行，维度为 $K$（而非 $V$）
- 词到槽位的映射是多对多的（一词多槽位，一槽位可能被多词共享）
- 解码时需要遍历词汇表，计算每个词的槽位平均分数

### F.11 V1 → V2 迭代演化

#### F.11.1 V1 的问题

V1 架构在实验中暴露了几个问题：

**问题 1：写入时的梯度稀释**

V1 使用 Softmax 词级分类，写入时将误差均分到目标词的所有槽位：

$$\Delta B[v, i, m] = \frac{\eta \cdot e_v \cdot \varphi[i, m]}{|S_v|}$$

这导致长词（槽位多）的每个槽位获得的梯度更小，学习效率低。

**问题 2：全局竞争的副作用**

Softmax 的全局竞争特性意味着：提高一个词的概率必然降低其他所有词的概率。在槽位空间中，这导致共享槽位的词之间产生不必要的竞争。

**问题 3：位置编码的过度区分**

V1 的 bigram + 位置编码使得同一 bigram 在不同位置映射到不同槽位。这虽然提供了位置敏感性，但也降低了形态相似词之间的槽位共享率。

#### F.11.2 V2 的改进

**改进 1：Sigmoid 多标签分类**

将词级 Softmax 替换为槽位级 Sigmoid：

| 方面 | V1 (Softmax) | V2 (Sigmoid) |
|------|-------------|--------------|
| 目标 | 预测词 | 预测槽位集合 |
| 竞争 | 全局（词之间） | 无（槽位独立） |
| 梯度 | 分散到所有词 | 集中在相关槽位 |
| 写入 | $\Delta B / |S_v|$ | $\Delta B$（不除） |

**改进 2：负样本抑制系数**

V2 对负样本的抑制强度乘以 0.1 系数：

$$e[j]_\text{neg} = -0.1 \times \sigma(l[j])$$

原因：
- 避免过度抑制共享槽位
- 正样本学习优先，负样本仅做轻微修正
- 实验验证：系数 1.0 导致学习不稳定，0.1 效果最佳

**改进 3：纯 N-gram 统计**

移除位置编码，使用纯 trigram 统计：

- 简化实现（无需微观 EMA）
- 提高形态相似词的槽位共享率
- N-gram 本身已蕴含局部顺序信息

**改进 4：频次加权 Q 更新**

V2 使用 N-gram 频次作为 Q 更新的刺激强度：

```python
# V1: 二值激活
Q[slots] += 1.0

# V2: 频次加权
for slot_id, count in slot_counts.items():
    Q_sparse[slot_id] += float(count)
```

这使得重复 N-gram 对 Q 的贡献更大，更好地反映词的内部结构。

**改进 5：稀疏 Q 存储**

V2 使用稀疏字典存储 Q，而非密集数组：

```python
# V1: 密集数组 (K, N)
self.Q = np.zeros((K, N))

# V2: 稀疏字典
self.Q_sparse = {}  # slot_id -> np.array(N,)
```

优势：
- 内存效率：只存储活跃槽位
- 计算效率：只处理非零条目
- 支持更大的 K（65536 vs 2048）

#### F.11.3 V2 的实验效果

| 指标 | V1 | V2 | 改进 |
|------|----|----|------|
| 形态泛化准确率 | ~60% | ~85% | +25% |
| 训练稳定性 | 波动大 | 稳定 | ✓ |
| 内存占用 | $O(K \cdot N)$ | $O(K_\text{active} \cdot N)$ | 降低 |
| 槽位共享率 (cat/cats) | ~30% | ~50% | +20% |

#### F.11.4 V2 的特性

**V2 的区分能力**：

V2 通过两种机制区分不同词：

1. **碎片集合差异**：不同词产生不同的 N-gram 集合
   - "Fly" → `^fl`, `fly`, `ly$`（3个槽位）
   - "FlyFlyFlyFly" → `^fl`, `fly`, `lyf`, `yfl`, `ly$`（5个槽位，多了桥接碎片）

2. **频次差异**：相同 N-gram 的出现次数不同
   - "Fly" 中 `fly` 出现 1 次
   - "FlyFlyFlyFly" 中 `fly` 出现 4 次，`lyf` 和 `yfl` 各出现 3 次

即使是重复字符的词也能区分：
- "aa" → `^aa`=1, `aa$`=1（总强度=2）
- "aaaa" → `^aa`=1, `aaa`=2, `aa$`=1（总强度=4，多了 `aaa` 碎片）

**槽位共享是设计目标，不是冲突**：

HashRetina 的核心目的就是让形态相似的词共享槽位：

| 词对 | 共享率 | 共享的 N-gram |
|------|--------|---------------|
| cat / cats | 40% | `^ca`, `cat` |

## 附录 G：能量守恒写入规则的演化历史（已废弃，仅供参考）

> **状态**：本附录记录了能量守恒写入规则的早期探索过程。其中的单向写入规则（$g = \max(0, 1-c)$）和 B-only 视角已被 §3.3–§3.4 的**统一突触双向能量守恒**取代。主要差异：
>
> 1. **单向 → 双向**：早期版本 $g = \max(0, 1-c)$ 只能正向补偿，无法修正 A 的 CP 过冲。现版本 $g = 1 - (s_A + s_B)$ 允许负向抑制，通过压缩映射保证收敛。
> 2. **B-only → 统一突触**：早期版本不感知 A 的存在，B 独立写入。现版本 B 的写入感知 A 的 per-connection 投影强度 $s_A$，只补偿 A 的残差。
> 3. **Sleep 协同**：双向写入的负修正通过 Sleep（$\text{target} = \hat{A}_\text{old} + B$）反馈给 A，逐轮收敛 A 的过冲。
>
> 最新规范见 §3.3（写入规则）、§3.4（数学性质）、§3.5（示例）。

### G.1 问题诊断：§3.3 Delta Rule 在非正交空间的失效

§3.3 定义的 Delta Rule 写入目标为 Dirichlet 分布 $p(v^*) \to 1$，误差信号 $e_{v^*} = 1 - p(v^*)$。这一设计在理论上具备自停止特性（$p \to 1 \Rightarrow e \to 0$），但在 Cosine Chaos 编码空间中存在根本性障碍：

**问题链**：

1. **φ 非正交性**：不同上下文的 $\hat\varphi$ 之间基线余弦相似度 $\approx 0.43$（实测），远非正交
2. **Dirichlet 目标不可达**：在非正交空间中，将某个 target 的 $p$ 推到 1.0 需要无限大的 logit 差异，模型永远追不上
3. **无限追逐**：$e_{v^*}$ 永远接近 1.0，等价于纯 Hebbian —— 自停止机制名存实亡
4. **高频压制**：高频词反复积累 $+\eta \cdot \varphi$，而 softmax 在 K=65536 的空间中对少数高分 slot 的概率集中效应使负抑制 $e_v = -p(v) \approx 0$ 对绝大多数 slot 无效

**根本原因**：Delta Rule 的目标（全局 Dirichlet 分布）是一个**绝对目标**，但在非正交 φ 空间中绝对目标不可达。需要一个**相对目标**。

### G.2 核心思想：per-connection 容量约束与自停止

**观察**：真正需要的不是"让目标的全局概率 $p(v^*) \to 1$"，而是"让每个 source slot 的连接中，正确 target 排第一即停"。

**设计原则**：

1. **per-connection 视角**：每个 $(s, t)$ 连接独立管理自身强度，不依赖全局 softmax
2. **容量守恒**：每个连接的"强度"（$\hat\varphi \cdot \mathbf{b}_{s \to t}$）有上界 1.0
3. **剩余容量驱动写入**：写入量 $\propto (1 - \text{当前强度})$，已强连接几乎停写
4. **非目标自然消退**：不显式抑制非目标连接，仅依赖稀疏衰减 $\lambda_B$（惰性执行）

### G.3 φ 归一化

为使"连接强度"可解释为 $[0, 1]$ 的投影系数，对 φ 进行 L2 归一化：

$$\hat\varphi_i = \frac{\varphi_i}{\|\varphi_i\|}$$

**归一化安全性（实验验证）**：

| 指标 | 数值 | 含义 |
|------|------|------|
| $\|\varphi\|$ 变异系数 CV | 0.15 | 范数几乎恒定，归一化 ≈ 常数缩放 |
| 归一化前后 cosine 差 | $< 10^{-6}$ | 方向信息完全保留 |
| 相邻上下文(dt≤3) cos | 0.88 | |
| 远距上下文(dt≥20) cos | 0.55 | |
| 近-远间距变化 | -0.0% | 归一化不影响区分度 |

### G.4 写入规则

定义连接强度：

$$c_{s \to t} = \hat\varphi_s \cdot \mathbf{b}_{s \to t}$$

**写入**（仅对真实 target $t = v^*$）：

$$\Delta \mathbf{b}_{s \to t} = \eta \cdot \max(0, \; 1 - c_{s \to t}) \cdot \hat\varphi_s$$

**稀疏衰减**（惰性执行，仅访问条目）：

$$\mathbf{b}_{s \to t} \leftarrow (1 - \lambda_B)^{\Delta t} \cdot \mathbf{b}_{s \to t}$$

其中 $\Delta t$ 为自上次访问以来的步数。衰减仅在条目被预测或写入时执行。

**预测**（使用归一化 φ）：

$$l_B[v] = \sum_{i \in \text{active}} \mathbf{b}_{i \to v} \cdot \hat\varphi_i$$

**关键性质**：

- **自停止**：$c \to 1 \Rightarrow \Delta b \to 0$，连接饱和后自动停写
- **无负样本**：不需要显式抑制非目标，降低计算开销
- **范数有界**：写入量 $\leq \eta$（首次）且递减，不会无限增长
- **均衡性**：高频词和低频词的连接都受同一个容量约束 $c \leq 1$

### G.5 自停止均衡分析

忽略衰减时，$c$ 的递推：

$$c_{n+1} = c_n + \eta(1 - c_n) \cdot \|\hat\varphi\|^2 = c_n + \eta(1 - c_n)$$

（因 $\|\hat\varphi\| = 1$）

$$c_n = 1 - (1 - \eta)^n$$

对 $\eta = 0.5$：$c_1 = 0.5, \; c_2 = 0.75, \; c_5 = 0.97, \; c_{10} = 0.999$

约 5 次写入后连接即达到饱和。加入衰减 $\lambda_B$ 后均衡点：

$$c^* = \frac{\eta}{\eta + \lambda_B} \approx 1 - \frac{\lambda_B}{\eta}$$

对 $\eta = 0.5, \lambda_B = 10^{-4}$：$c^* \approx 0.9998$。

### G.7 已知问题与待深入分析

1. **连接强度溢出**：部分连接的 $c > 1$（实测 max ≈ 1.4）。原因：写入时和读取时的 $\hat\varphi$ 方向不同（上下文在变化），多次在略有偏转方向上的写入可累积超过 1.0。需要研究是否需要显式 clip 或其他补偿。

2. **投票稀释**：随序列推进，活跃 source slot 增多（pos=5 时 22 个，pos=150 时 276 个），但正确投票 source 占比从 50% 降至 9%。大量"无关 source"的噪声投票会稀释正确信号。

3. **target-other 间距收窄**：pos=5 时 target-other 强度间距 0.86，pos=150 时仅 0.28。深入分析间距收窄的机制和改善方案。

4. **per-source 总能量约束**：当前实现是 per-connection 容量 ≤ 1，一个 source 连接到多个 target 时总能量可超过 1。是否需要 per-source 总能量归一（连接间竞争）有待探索。

5. **与 Sleep 巩固的兼容性**：Energy 写入产生的 B 张量特性（范数 ≈ 1, 方向对齐 $\hat\varphi$）是否适合现有的 Sleep 采样与 A 巩固流程。

## 附录 H：并行训练与批量处理

### H.1 动机与挑战

串行训练的瓶颈：对于大规模语料（百万至千万句），串行训练每次只处理一个样本，无法充分利用现代 GPU 的并行计算能力。

**核心挑战**：如何在保持 Energy 守恒规则的前提下实现批量并行训练？

传统的批量梯度下降（BGD）直接对梯度求和，但这会破坏 MathBrain 的能量守恒机制：
- 如果 50 个样本都更新同一个连接，直接相加相当于学习率 × 50
- 会导致范数爆炸，破坏 $s_A + s_B \to 1$ 的收敛性

### H.2 并行共识合并策略（Parallel Consensus Merge）

**核心思想**：Batch 内所有样本看到同一个 frozen B，各自计算更新量，对共享连接取**均值**而非求和。

#### H.2.1 算法流程

**步骤 1：并行前向与误差计算**

对 Batch 内的每个样本 $i$，使用同一个 frozen 的 B 矩阵，并行计算：

$$g^{(i)} = 1.0 - (s_A^{(i)} + s_B^{(i)})$$

$$\Delta \mathbf{b}^{(i)}_{\text{raw}} = \eta \cdot g^{(i)} \cdot \hat{\varphi}^{(i)}$$

其中：
- $s_A^{(i)} = \hat{\varphi}^{(i)} \cdot A[\text{src}, \text{tgt}]$：A 的投影强度
- $s_B^{(i)} = \hat{\varphi}^{(i)} \cdot B[\text{src}, \text{tgt}]$：B 的投影强度（使用 frozen B）

**步骤 2：按连接进行共识平均**

对于每个连接 $(\text{src}, \text{tgt})$，收集 Batch 内所有指向它的更新请求，取**均值**：

$$\Delta \mathbf{b}_{\text{merged}}[\text{src}, \text{tgt}] = \frac{1}{n} \sum_{i=1}^{n} \Delta \mathbf{b}^{(i)}_{\text{raw}}$$

其中 $n$ 是 Batch 内更新该连接的样本数。

**步骤 3：应用合并更新**

$$B[\text{src}, \text{tgt}] \leftarrow B[\text{src}, \text{tgt}] + \Delta \mathbf{b}_{\text{merged}}[\text{src}, \text{tgt}]$$

#### H.2.2 物理意义

**为什么取均值而不是求和？**

1. **防止高频词爆炸**：假设 "the → dog" 在一个 Batch 里出现 50 次。因为它们看到的 frozen B 相同，计算出的 gap 基本相同。取均值后，它获得的更新量 $\approx 1 \times \eta \cdot \text{gap}$，等价于串行看了一次。

2. **正交抵消噪音**：如果这 50 次更新对应的上下文方向 $\hat{\varphi}$ 各不相同（哈希碰撞），取均值后，张量会自动发生正交抵消，噪音被抹平，只有真正的共识方向被保留。

3. **保持 O(1) 受力**：通过取均值，将原本 $O(n)$ 增长的推力强制钳制回 $O(1)$ 的单步物理受力区间，保持 Energy 守恒规则。

#### H.2.3 负写入的关键作用

允许 $g < 0$ 产生负写入是并行训练的**保命机制**：

- 在串行训练中，每次更新步长很小，不容易过冲
- 在并行训练中，如果同一连接在 Batch 内出现多次，即使取均值也可能累积过冲
- **负写入提供自动纠错**：下一个 Batch 发现 $s_A + s_B > 1.0$，立即产生 $g < 0$，释放反向抑制力，将过冲拉回 1.0

这赋予模型**极强的自我纠错与抗并行震荡能力**。

### H.5 实现要点

#### H.5.1 B 内存的并行写入接口

```python
def write_batch(self, batch_data: List[Dict], a_knowledge=None):
    """Batch 写入（并行共识合并）
    
    Args:
        batch_data: List of dicts, each containing:
            - 'target_slots': Dict[int, int] 目标槽位
            - 'active_slots': np.ndarray 活跃槽位
            - 'phi': np.ndarray phi 向量
        a_knowledge: A 知识（用于计算 A 强度）
    """
    # 收集所有写入请求
    write_requests = defaultdict(list)  # (src, tgt) → [(phi_hat, gap), ...]
    
    # 步骤 1：并行计算所有 gap
    for item in batch_data:
        for src, tgt in connections:
            # 计算 B 强度和 A 强度
            s_B = phi_hat @ B[src][tgt]
            s_A = phi_hat @ A[src][tgt]
            gap = 1.0 - (s_A + s_B)
            
            write_requests[(src, tgt)].append((phi_hat, gap))
    
    # 步骤 2：对共享连接取均值
    for (src, tgt), requests in write_requests.items():
        if len(requests) == 1:
            phi_hat, gap = requests[0]
            delta = eta * gap * phi_hat
        else:
            # 共享：取均值
            phi_mean = mean([phi for phi, _ in requests])
            gap_mean = mean([gap for _, gap in requests])
            delta = eta * gap_mean * phi_mean
        
        # 应用更新
        B[src][tgt] += delta
```

#### H.5.2 与预处理的结合

并行训练可以与预处理（§附录 I）结合，实现最大化加速：

1. **全局预处理**：预计算所有句子的 Q 状态序列和 φ 编码
2. **Batch 训练**：使用预处理数据进行并行训练
3. **总加速比**：预处理 2x × 并行 3-7x = **6-14x 总加速**

### H.6 理论保证

**定理**：并行共识合并策略保持 Energy 守恒规则的收敛性。

**证明要点**：

1. 对于非共享连接（Batch 内只有 1 个样本更新），更新量与串行完全相同
2. 对于共享连接（Batch 内有 $n$ 个样本更新），均值更新等价于：
   - 看到 $n$ 个样本的"平均上下文"
   - 在该平均上下文下进行一次串行更新
3. 由于 $g = 1 - (s_A + s_B)$ 是压缩映射，无论更新顺序如何，最终都收敛到 $s_A + s_B = 1$
4. 负写入机制保证即使出现暂时过冲，后续 Batch 也会自动纠正

**推论**：增加训练轮次可以让大 Batch 达到与串行相同的准确度。

### H.7 与传统 Mini-Batch SGD 的对比

| 特性 | Mini-Batch SGD | MathBrain 并行共识 |
|------|---------------|-------------------|
| 梯度合并 | 求和 | **均值** |
| 学习率调整 | 需要随 Batch 增大而调整 | **无需调整**（均值自动归一化） |
| 高频样本 | 容易主导梯度 | **自动平衡**（均值抑制高频） |
| 过冲处理 | 依赖学习率衰减 | **负写入自动纠错** |
| 收敛保证 | 需要精心调参 | **压缩映射保证收敛** |

MathBrain 的并行策略更加鲁棒，无需复杂的学习率调度。
